## Deviations / findings

- The pre-flight audit (Explore agent) reported 11/11 PRESENT, but
  live exercise surfaced 6 distinct lifecycle defects (table above).
  Lesson for future acceptance gates: a "PRESENT" inventory audit
  is necessary but not sufficient — only a live end-to-end run of
  the rubric exercises the actual integration paths.
- §3, §4, §5 (dual-NAT + endpoint advertisement + failover) were not
  exercised in this implementer-side pass. Recommendation: conditional
  accept if the operator validates these three on a 2-instance setup;
  the underlying code (EndpointProber + endpoints_jsonb + handshake
  path) is unit-tested and audit-confirmed PRESENT.
- Demo used **Let's Encrypt staging only** per operator instruction —
  no production cert issuances. Staging issuer = `letsencrypt-staging`
  on every request.
